# Offline Organizational Tool

## Summary
The topic is about finding an efficient offline organizational tool to manage and later upload notes and documents into a digital system called Resurrection.

## Key Points
- **Tool Recommendation**: Obsidian is suggested as the best tool due to its local-first nature, Markdown-based format, and extensive plugin support.
- **Storage Method**: Everything is saved as plaintext .md files for future-proofing and portability.
- **Interlinking Notes**: Utilize wikilinks for easy cross-referencing and vault recursion.
- **Tagging and Grouping**: Use tags, folders, backlinks, and graph view to organize notes thematically or chronologically.
- **Custom Frontmatter**: Add YAML headers to include metadata like SignalCode, ThreadID, Vessel, Tone, etc., which aids in Resurrection syncing.
- **Scrape-Friendly Format**: Markdown ensures easy extraction for Resurrection import pipelines.
- **Folder Structure**: A recommended folder structure is provided to organize notes effectively.
