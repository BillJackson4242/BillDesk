# Agent Protocol Auditlayer Definitions

## Summary
Agent Protocol Auditlayer Definitions explain key components of the Vault system, including Agents (semi-autonomous entities with specific functions), Protocols (rule-based behavior systems), and Background Audit Layers (permanent monitoring systems).

## Key Points
- **AGENT**: 
  - Semi-autonomous, personified entity with a defined role.
  - Manifests visually through an avatar or tile.
  - Behaves differently based on the assigned function.
  - Memory is needed for persistence but can be simulated via external systems.

- **PROTOCOL**:
  - Rule-based behavior system that does not "speak" unless dictated by rules.
  - Silent in its operation, altering responses and injecting warnings or formatting.
  - Can be always active or triggered by commands, uploads, or prompts.
  - Configurable through text, Markdown, prompt syntax.

- **BACKGROUND AUDIT LAYER**:
  - Permanent monitoring system that reviews actions and behaviors.
  - No specific details provided in the excerpt but implied to be a crucial component of the Vault system's oversight mechanisms.
