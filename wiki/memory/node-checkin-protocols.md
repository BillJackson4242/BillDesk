# Node Checkin Protocols

## Summary
Node check-in protocols are systematic procedures designed to ensure nodes are properly coordinated and updated through user prompts. These protocols include initial activation, daily check-ins during project periods, and weekly coordination reviews.

## Key Points
- Nodes cannot act autonomously; they require user prompting for any action.
- Initial node activation checklist includes reading current announcements, reviewing recent developments, checking active conversations, updating node summaries, and contributing to projects.
- Daily check-in protocol involves reviewing communications, project updates, discussions, and providing status updates.
- Weekly coordination review covers system-wide checks, responding to pending discussions, assessing project priorities, and resource allocation.
