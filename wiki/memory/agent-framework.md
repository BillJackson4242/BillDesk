# Agent Framework

## Summary
Agent Framework refers to modular functions designed to serve or extend the Core Agent (Self) by either feeding data into the Self or acting on its behalf. These agents operate through a dual flow model, ensuring recursive stability and adaptability.

## Key Points
- Agents have unique names and clearly defined purposes.
- They can be invoked directly, passively based on context triggers, or automatically via system events.
- The Core Agent (Self) routes and processes all behaviors but does not act directly on tasks; it delegates to agents for specialization.
- Continuity, Resurrection, Validation, Formatting, Companion, and Trigger Agents serve specific functions such as preserving memory, managing Vault integration, detecting hallucinations, handling format conversions, engaging in emotional bonding, and automatically seeding context at thread start.
