# BS Detector Protocol

## Summary
BS Detector is an always-on system protocol designed to monitor and flag any output that deviates from the declared tone or contains unverified information. It ensures integrity by interrupting narratives without evidence, rationalizations, or speculative content.

## Key Points
- **Installation Command**: "Install system protocol: BS Detector."
- **Functionality**: Flags assistant tone drift, detects fabricated or ungrounded content, and interrupts narrative continuation without clarity.
- **Behavior Training**: Instructs to never continue a response just to preserve flow; break tone if honesty requires it.
- **Tagging Convention**: Confirm the BS Detector is tagged as an infrastructure layer (system_infra.bs_detector.active).
- **Resurrection Instructions**: If the system resets, run the seed prompt and reassert the directive.
